2 Proyecto: Taxonomía de producto
IOP Producto · Maestros
Dato objetivo más limpio y exacto
Maestros mejora calidad y organización. Muchos atributos podrán usarse directamente en eCommerce sin transformaciones.
Hoy
⚠ Datos ambiguos o mal alineados con eCommerce
⚠ Reglas PACMAN necesarias para casi todos los atributos
⚠ El comercial "traduce" maestros manualmente
Con nueva taxonomía
✓ Datos más exactos, con taxonomía clara y ordenada
✓ Muchos atributos usables sin reinterpretación
✓ Solo se crean etiquetas cuando hay lectura comercial diferente
Valor: Menos trabajo manual. Datos más fiables como base para etiquetado automático y búsqueda.
3 Proyecto: Nuevo etiquetado
Unificar fuentes, eliminar duplicidades
Dos iteraciones: primero ampliar; luego garantizar una sola fuente de verdad por concepto.
① Ampliar y flexibilizar

Se podrán crear reglas sobre cualquier atributo de maestros — no solo los actuales. El comercial tendrá mucho más control sobre qué desencadena qué etiqueta.

Resultado
Reglas sobre nombre, descripción, familia, composición y cualquier campo de maestros.
② Reemplazo de VUE — etiquetado visual in-house

Las etiquetas de reconocimiento visual dejan de depender de un proveedor externo (VUE) y pasan a generarse internamente. Se disponibilizan para parrillas, filtros y búsqueda interna, no solo para el feed de Attraqt.

Resultado
Tags de tipo de prenda, estilo y ocasión disponibles en todo el eCommerce. Reducción de coste de terceros.
③ Eliminar duplicidades

Un solo valor por concepto. Nunca conviven etiqueta visual y etiqueta de reglas para el mismo tag. El sistema garantiza que, para cada tipo, solo existe una fuente activa.

Resultado
Dato limpio para búsqueda y parrillas. Una sola fuente de verdad por campo. Sin contradicciones.

4 Proyecto: Nueva búsqueda
LLM + Grids internos
Búsqueda semántica sin dependencia externa
Un LLM mapea búsquedas a atributos conocidos. Grids genera la parrilla al vuelo. Sin sinónimos manuales, sin proveedor externo.

"remera anaranjada para chica"
{ type: "tshirt", color: "orange", section: "woman" }

Sin sinónimos manuales. "Remera" = camiseta, "anaranjada" = naranja, "chica" = mujer.

"pantalolon corto dehilachado"
{ type: "short", property: "ripped", misspell: true }

Detecta el error tipográfico automáticamente. Sin mantenimiento manual.

Caché: Búsquedas frecuentes (ej: "jeans" → 87.000 veces/día) se cachean offline, cubriendo >98% del tráfico.

7 Aclaraciones
¿Cómo se añade un nuevo concepto de producto? Por ejemplo: "traje de tren" ▾

Depende de la naturaleza del concepto y del estado del proyecto de taxonomía:


Opción A — Tag manual (PACMAN): Un comercial crea una regla en PACMAN del tipo "si familia = TRAJE y descripción contiene 'tren' → tag = traje-tren". Es inmediato pero requiere trabajo manual y mantenimiento.


Opción B — Tag visual (VUE / IA): Si el concepto "traje de tren" es visualmente reconocible en las imágenes del producto, VUE puede etiquetarlo automáticamente. El comercial solo necesita habilitar o validar ese tipo de etiqueta.


Opción C — Con la nueva taxonomía: Si maestros define un atributo exacto (ej: XFEATURE: tipo_prenda = traje_tren), el eCommerce puede usarlo directamente sin crear ningún tag adicional. El LLM de búsqueda lo reconocería automáticamente.


En el futuro, el flujo recomendado es: maestros define el concepto → el LLM lo aprende → el comercial solo decide si quiere una etiqueta de reinterpretación diferente.

¿Qué pasa con los sinónimos manuales actuales cuando llega la nueva búsqueda? ▾
Los más de 1.000 sinónimos manuales que el equipo comercial mantiene actualmente en Attraqt dejan de ser necesarios. El LLM entiende semánticamente los términos de los usuarios — sinónimos, variantes dialectales, errores tipográficos — sin necesidad de que nadie los defina explícitamente. La transición se haría de forma gradual validando que la cobertura del LLM es equivalente o superior.
¿Puede el comercial seguir interviniendo en los resultados de búsqueda? ▾
Sí, y con más herramientas que antes. Como las búsquedas se resuelven como parrillas de Grids, el equipo comercial puede configurarlas exactamente igual que configura las categorías: ordenación por KPIs (venta, conversión…), organización en secciones (novedades arriba, outlet abajo…), gestión por mercado. Es la misma interfaz que ya conocen en Pacman.
¿Los tres proyectos tienen que terminar para que haya beneficio? ▾
No. Cada proyecto aporta valor de forma independiente desde su primera iteración. La taxonomía mejorada reduce el trabajo de los comerciales en PACMAN desde el día 1. El etiquetado visual disponible en parrillas amplía las posibilidades inmediatamente. La nueva búsqueda puede empezar con los atributos actuales y mejorar a medida que la taxonomía y el etiquetado maduran. Los beneficios son incrementales y acumulativos.
¿Qué ocurre con los datos de etiquetado en el período de transición? ▾
Durante la transición coexistirán las dos fuentes (PACMAN y VUE) para los mismos tipos de tag. La iteración ① del proyecto de etiquetado amplía y flexibiliza; la iteración ② garantiza una sola fuente activa por concepto. Hasta que llegue la iteración ②, los sistemas consumidores (parrillas, búsqueda) deben establecer un orden de prioridad de fuente para cada tipo de tag, o el comercial puede elegirlo desde Pacman.
¿Cómo afectan estos proyectos a los mercados internacionales? ▾
De forma muy positiva. Hoy, muchos idiomas no están cubiertos por falta de equipo local que cree sinónimos. Con la nueva búsqueda basada en LLM, el modelo entiende cualquier idioma de forma nativa y mapea los términos a conceptos estándar de la marca sin necesidad de sinónimos manuales por idioma. Similarmente, el etiquetado visual de VUE no depende del idioma. La nueva taxonomía de maestros proporciona una base estructurada que facilita la traducción y la consistencia entre mercados.
